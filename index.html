Vision & Mission

Our Vision

To build a generation of engineers who don't just write code — they think in systems, break boundaries, and architect the future. Code Crypt exists to transform curious students into confident technologists who lead, innovate, and inspire.

  • Foster a culture of curiosity and relentless learning
  • Bridge the gap between academics and industry
  • Make our campus a launchpad for tech innovators

Our Mission

To create a vibrant, inclusive community where every student — from a first-year beginner to a final-year expert — can collaborate, compete, and grow. We do this through hands-on events, open-source culture, and mentorship from peers and professionals.

  • Organize impactful workshops, hackathons & CTFs
  • Mentor juniors through real-world project experience
  • Connect members with alumni and industry partners

Flash Challenges

Week 1 — Blueprint Blitz

Submit a high-quality Architecture Diagram: UI/UX wireframe (software) or circuit schematic (hardware)

Week 2 — Code Kickoff

Initialize GitHub repo + first significant commit with a professional README.md

Week 3 — Teaser Trailer

Post a 30–60 sec teaser on LinkedIn/Instagram tagging Code Crypt & ACM-W Stanley

Week 4 — Documentation Sprint

Submit first draft of Technical Documentation including Problem Statement ID, SDG, and tech stack

Guidelines

  • Documentation: All projects must clearly state which SDG they address. Documentation must follow the official blueprint but can be presented creatively.
  • GitHub Repository: All code, circuit designs, and documentation must be uploaded to a public GitHub repository upon completion.
  • Certification: MVP Team certificates awarded per domain based on live demo, documentation quality, and GitHub submission.
  • Component Sourcing: The college/club will not provide hardware components or kits. Teams must source their own materials.
  • Infrastructure: Basic power sockets provided at venue. Teams are strongly encouraged to bring their own extension boards/spike busters.
  • Safety & Compliance: All hardware must be safe and properly insulated. High-voltage components or flammable materials require prior faculty approval.
